# Your first login

Once SuperToolMake is running, open the builder at
[http://localhost:10000/builder](http://localhost:10000/builder) (replace
`localhost:10000` with your own host and port if you've changed them).

## Creating an admin user

The very first time you visit the builder you'll be prompted to create an admin
user. The admin user has access to everything.

![Create an admin user](/img/screenshots/super_admin.png)

Enter an email address and a password, confirm the password, then select
**Create admin user**. You'll be logged in and taken to the SuperToolMake portal.

:::warning
Keep these credentials safe — the admin account can manage every workspace, user
and role in your instance.
:::

## The portal

After logging in you'll land on the portal. From here you can:

- Open the workspaces and apps you have access to.
- Create a new workspace to start building.

Builders and admins can create workspaces; end users only see the apps they have
been granted access to.

## Creating your first workspace

A **workspace** is the container for everything you build: your data connections,
your apps, your roles and your users. Create one, give it a name, and you'll be
taken into the builder where you can:

1. **[Connect a SQL database](../data/connecting-sql.mdx)** or
   **[an API](../apis/overview.mdx)** to bring in your data.
2. **[Design an app](../apps/overview.mdx)** with screens and forms.
3. **[Set up access control](../access-control/roles.mdx)** for your end users.
4. **[Publish](../apps/publishing.mdx)** your app so end users can use it.
