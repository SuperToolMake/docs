# Views

A **view** is a saved, reusable perspective on a table. Views let you present a
subset of your data — with specific columns, filters, and sorting — without changing
the underlying table.

## Why use views

- **Show only what's relevant** — hide columns that end users don't need.
- **Pre-filter data** — for example, a "Active customers" view that only shows rows
  where `Status = Active`.
- **Apply default sorting** — order rows by a column such as a date or name.
- **Scope access** — grant a role access to a view rather than the whole table, so
  users only see the rows and columns you intend.

## Creating a view

From a table in the **SQL tables** section, create a new view and then configure it:

1. **Choose visible columns** — select which fields appear, and reorder them.
2. **Add filters** — define the conditions a row must meet to appear.
3. **Set sorting** — pick a column and direction.

## Using views in apps

Views can be used anywhere a table can be used as a data source — for example in
[data providers](../apps/components.mdx), tables, and
[autogenerated screens](../apps/screens.mdx). Pointing a screen at a view (instead of a
raw table) is a simple way to control exactly what end users can see.
